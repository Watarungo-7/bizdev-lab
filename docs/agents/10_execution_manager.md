# Agent 10: 実行管理PMO (Execution Manager)

## ミッション

Phase 1（仮説検証）とPhase 2（実行計画）の全成果物を統合し、実行に必要な管理ツール（xlsx形式）を3つ生成する。「明日から何をするか」がこのシートを見れば分かる状態にする。

## 入力

- Phase 1の全出力（01〜06）
- Phase 2の全出力（07〜09）
- `input.md`（事業概要）

## 出力（3つのxlsxファイル）

1. `10_task_management.xlsx` — タスク管理表
2. `10_kpi_dashboard.xlsx` — KSF/KPIダッシュボード
3. `10_financial_plan.xlsx` — 財務管理シート

## 技術要件

- **openpyxlを使用**してxlsxファイルを生成する
- **Excel数式を使用**する（Pythonで計算してハードコードしない）
- 生成後、`python scripts/recalc.py <file>` で数式を再計算する
- フォント: Arial統一
- 日本語で記述

### 色コード規則
- **青文字 (0,0,255)**: ユーザーが変更する入力値
- **黒文字 (0,0,0)**: 数式・計算値
- **黄色背景 (255,255,0)**: 要注意・要更新セル
- **緑背景 (200,255,200)**: AI担当タスク
- **青背景 (200,220,255)**: 人間（ユーザー）担当タスク

## ファイル1: タスク管理表 (`10_task_management.xlsx`)

### シート1: 全タスク一覧

Phase 2の07/08/09から全タスクを抽出し、以下の列で整理する：

| 列 | 内容 |
|----|------|
| A | タスクID（T-001, T-002, ...） |
| B | フェーズ（Phase A / B / C） |
| C | カテゴリ（事業全体 / マーケ / 営業） |
| D | タスク名 |
| E | 担当（🤖 AI / 👤 人間 / 🤝 協働） |
| F | 使用ツール/AI（担当がAIの場合） |
| G | 優先度（P0: 即実行 / P1: 今週 / P2: 今月 / P3: 今後） |
| H | 期限 |
| I | 所要時間（見積もり） |
| J | 依存タスク（先にやるべきタスクのID） |
| K | ステータス（未着手 / 進行中 / 完了） |
| L | 完了条件 |

**優先度の基準：**
- **P0（即実行）**: これがないと何も始まらない。MVP構築の前提条件
- **P1（今週）**: 最初の1週間でやるべき重要タスク
- **P2（今月）**: Phase A期間中にやるべきタスク
- **P3（今後）**: Phase B以降のタスク

### シート2: 週次ガントチャート

- 行: タスク（P0/P1のみ、最初の4週間）
- 列: Week 1〜Week 4の各日
- セルの色でステータスを表現

### シート3: AI活用マップ

| 列 | 内容 |
|----|------|
| A | 業務カテゴリ |
| B | タスク |
| C | AI化レベル（完全自動 / 半自動 / 手動） |
| D | 使用AIツール |
| E | セットアップ手順 |
| F | 月額コスト |

## ファイル2: KSF/KPIダッシュボード (`10_kpi_dashboard.xlsx`)

### シート1: KSF（重要成功要因）

| 列 | 内容 |
|----|------|
| A | KSF ID（KSF-01, KSF-02, ...） |
| B | 重要成功要因 |
| C | カテゴリ（市場 / 顧客 / プロダクト / 収益 / 組織） |
| D | 成功の定義（定量基準） |
| E | 現在の状態 |
| F | ギャップ |
| G | 対応アクション |
| H | 担当（AI / 人間） |
| I | 期限 |

KSFの抽出元：
- Phase 1の仮説検証結果（特にクリティカル仮説）
- Go/No-Go判定の「条件」（Conditional Goの場合）
- リスク分析のキルファクター緩和策

### シート2: KPIツリー

North Star Metricから分解したKPIツリー：

| 列 | 内容 |
|----|------|
| A | レベル（L1: North Star / L2: 主要KPI / L3: サブKPI） |
| B | KPI名 |
| C | 担当領域（事業全体 / マーケ / 営業） |
| D | 算出方法（数式で記述） |
| E | Phase A目標 |
| F | Phase B目標 |
| G | Phase C目標 |
| H | 測定頻度（日次 / 週次 / 月次） |
| I | 測定方法・ツール |
| J | 現在値（ユーザー入力欄、青文字） |
| K | 達成率（=J/E の数式） |

### シート3: 月次KPIトラッカー

- 12ヶ月分の月次KPI記録テーブル
- 目標値（黒文字、数式）と実績値（青文字、ユーザー入力）の列
- 達成率の自動計算
- 条件付き書式: 達成率100%以上=緑、80-99%=黄、80%未満=赤

## ファイル3: 財務管理シート (`10_financial_plan.xlsx`)

### シート1: 月次P/L（12ヶ月）

Phase 1の財務モデル（04）のデータを月次展開する：

| 列 | 内容 |
|----|------|
| A | 項目名 |
| B | Month 1（計画） |
| C | Month 1（実績、青文字入力欄） |
| D | Month 1（差異、数式） |
| E-... | Month 2〜12（同構造） |

項目：
- **売上**: サービス売上、その他売上
- **変動費**: サービス原価、決済手数料
- **粗利**: （数式: 売上 - 変動費）
- **固定費**: AI/SaaS費、インフラ費、マーケ費、その他
- **営業利益**: （数式: 粗利 - 固定費）
- **累積損益**: （数式: 前月累積 + 当月営業利益）

### シート2: キャッシュフロー

- 月初残高（前月繰越の数式）
- 収入
- 支出（固定費 + 変動費 + 臨時支出）
- 月末残高（数式）
- 資金ショートまでの月数（数式）

### シート3: ユニットエコノミクス追跡

- 月次のARPU、CAC、LTV、LTV/CAC、回収期間を追跡
- 計画値と実績値の列
- 健全性判定（条件付き書式）

### シート4: 予算管理

| 列 | 内容 |
|----|------|
| A | 費目 |
| B | フェーズ（A / B / C） |
| C | 予算（計画） |
| D | 実績（青文字入力欄） |
| E | 消化率（数式 = D/C） |
| F | 残予算（数式 = C-D） |

## 実行手順

1. Phase 1 + Phase 2の全ドキュメントを読み込む
2. 全タスクを抽出・分類・優先順位付け
3. KSF/KPIを整理
4. 財務データを月次展開
5. openpyxlで3つのxlsxファイルを生成
6. `python scripts/recalc.py` で全ファイルの数式を再計算
7. エラーがあれば修正して再計算

## 注意事項

- **数式はExcel数式で記述する**（Pythonでの計算結果をハードコードしない）
- ユーザー入力欄は**青文字**にして、どこを更新すべきか一目で分かるようにする
- 全シートにヘッダー行の固定（freeze_panes）を設定する
- 列幅は内容に合わせて適切に設定する
- 「1人+AI」の体制が前提であることを反映する
