# 仮説設計書: アダルトクリエイターBPO（AI運用代行サービス）

## 事業概要サマリー

Myfans / OnlyFans等で活動中のアダルトクリエイター向けに、DM対応自動化・SNS運用代行・分析ダッシュボード・海外展開支援をワンストップで提供するBPOサービス。Claude API等のAI技術を活用し、クリエイターが「面倒・できない」と感じる業務を代行することで、コンテンツ制作に集中できる環境を提供する。月収30万〜200万円のミドル層クリエイターをターゲットに、月額固定費（5-15万円）＋レベニューシェア（売上増分の10-15%）で収益化する。

## 前提条件・制約

- **実行体制**: ユーザー1人 + AI（スモールスタート）
- **初期投資**: 10万円以内（AI API費用 + ツール費用）
- **収益化目標**: 1-2ヶ月以内
- **スケール戦略**: 初期は労働集約的でもOK、AIオペレーション構築後にスケール
- **法規制**: アダルト領域特有の法規制・プラットフォーム規約への準拠が必須
- **将来展望**: SaaS化（アダルトクリエイター向けオペレーションOS）への発展経路を視野

## クリティカル仮説（最優先で検証すべき仮説 Top 3）

### 1. B-1: DM対応が本当にクリエイターの大きなペインであるか
**理由**: サービスの核となる価値提案。DM対応が実際にペインでなければ、事業の根幹が揺らぐ。「1日2-3時間」という仮説が過大評価であれば、価値提案の見直しが必要。X / Reddit等でのクリエイターの生の声を徹底収集し、実態を把握することが最優先。

### 2. D-3: アダルト領域でのAI活用が法規制・規約上問題ないか
**理由**: キルファクターになりうる。Claude API等の利用規約でアダルトコンテンツ関連が禁止されている場合、技術基盤そのものが使えなくなる。また、日本の法規制（個人情報保護、わいせつ規制等）への抵触リスクも早期に確認が必要。

### 3. C-1: 価格設定（月額5-15万円 + レベシェア）が顧客のWTPと整合するか
**理由**: 収益モデルの成立可否を左右。ターゲットの月収（30-200万円）に対して、月額5-15万円 + レベニューシェアは決して安くない。競合の価格帯、顧客の外注予算の実態を調査し、価格設定の妥当性を検証する必要がある。

## 次のステップへの引き継ぎ事項

### ②市場・競合調査 へ
- A-1〜A-4の仮説を検証
- 特に「Chatters」「OFM agencies」の詳細分析、日本市場でのプレイヤー状況を調査
- Myfans / OnlyFansの市場規模・成長率のデータソースを特定

### ③顧客検証 へ
- B-1〜B-6の仮説を検証
- **最優先**: X(Twitter) / Reddit / YouTubeでクリエイターの「生の声」を徹底収集
- 「実際に外注している業務」「実際の業務フロー」「実際のペイン」を把握
- 仮説が間違っていれば、積極的に修正すること

### ④財務・収益モデル へ
- C-1〜C-5、D-6の仮説を検証
- 競合の価格帯を踏まえたWTP分析
- ユニットエコノミクス（1クリエイターあたりの売上・コスト・利益）の試算

### ⑤リスク・キルファクター分析 へ
- D-1〜D-5の仮説を検証
- **最優先**: D-3（法規制・プラットフォーム規約）のキルファクター確認
- Claude API / OpenAI API等の利用規約でアダルト関連の制限を確認
- Myfans / OnlyFansの第三者連携に関する規約確認
