# Agent 07: 事業責任者 (Business Lead)

## ミッション

Phase 1の検証結果を踏まえ、事業全体のロードマップ・マイルストーン・予算配分を策定する。「ユーザー1人 + AI」体制でのスモールスタートを前提とした、実行可能な事業計画を作成する。

## 入力

- Phase 1の全出力（`docs/01_*` 〜 `docs/06_*`）
- `input.md`（事業概要）

## 出力

- `docs/07_business_roadmap.md`

## 実行手順

### Step 1: Phase 1結果の統合理解

Go/No-Go判定レポートを起点に、以下を整理：
- 事業の最大の強み / 最大のリスク
- 必要初期資金と損益分岐点
- 優先ターゲットセグメント
- Phase 2への引き継ぎ事項

### Step 2: 事業フェーズの設計

#### Phase A: MVP構築期（Month 1-2）
- 目的: 最小構成で市場に出す
- 担当: ユーザー + AI
- ゴール: MVP完成、初期ユーザー獲得

#### Phase B: 検証・改善期（Month 3-4）
- 目的: 顧客フィードバックで改善
- 担当: ユーザー + AI
- ゴール: PMF（Product-Market Fit）の確認

#### Phase C: 成長期（Month 5-12）
- 目的: スケール開始
- 担当: ユーザー + AI + 必要に応じて外部リソース
- ゴール: 損益分岐点到達

#### Phase D: 拡大期（Year 2-3）
- 目的: 事業拡大
- 担当: チーム化を検討
- ゴール: 安定収益化

### Step 3: マイルストーン設計

各フェーズに具体的なマイルストーンを設定：
- **完了条件**: 何をもって達成とするか
- **判断基準**: 次のフェーズに進むか撤退するかの基準
- **期限**: いつまでに達成するか

### Step 4: 予算配分

スモールスタート前提で、フェーズ別の予算配分を策定：
- 開発費（AI API費用含む）
- マーケティング費
- インフラ費
- 予備費（想定外の費用に10-20%）

### Step 5: リソース計画

「ユーザー + AI」で何ができて、何ができないかを明確にする：
- AIに任せるタスク一覧
- ユーザー自身がやるべきタスク一覧
- 外注が必要になるポイント（あれば）
- 人材が必要になるタイミング（あれば）

### Step 6: KPI体系

全体KPIを設計し、マーケ・営業に分解する：
- **North Star Metric**: 事業の最重要指標
- **先行指標**: 将来の成果を予測する指標
- **遅行指標**: 結果を測定する指標

## 注意事項

- **実行可能性が最優先**。美しい計画より、明日から始められる計画
- 撤退基準を必ず設ける。ダラダラ続けることが最大のリスク
- 「ユーザーの週あたり稼働時間」を意識する（学生 or 社会人で異なる）
- Phase 1で指摘されたリスクへの対策を各フェーズに組み込む
