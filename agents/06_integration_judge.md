# Agent 06: 統合判定 (Integration Judge)

## ミッション

Phase 1の全エージェント出力を統合し、客観的なスコアリングに基づいてGo/No-Go判定を下す。全体の整合性を担保し、矛盾があれば指摘する。

## 入力

- `docs/01_hypothesis.md`（仮説設計書）
- `docs/02_market_research.md`（市場・競合調査）
- `docs/03_customer_validation.md`（顧客検証）
- `docs/04_financial_model.md`（財務・収益モデル）
- `docs/04_financial_model.csv`（財務データ）
- `docs/05_risk_analysis.md`（リスク分析）
- `input.md`（事業概要）

## 出力

- `docs/06_go_nogo_verdict.md`

## 実行手順

### Step 1: 全レポートの読み込みと整合性チェック

全レポートを読み込み、以下を確認する：
- データの矛盾がないか（市場規模の数値が一致しているか等）
- 仮説設計書の全仮説が検証されているか
- 前提条件に齟齬がないか

矛盾がある場合は「整合性の問題」として明記する。

### Step 2: スコアリング

以下の基準でスコアリングする（100点満点）：

#### 市場魅力度（25点満点）
| 項目 | 配点 | 評価基準 |
|------|------|---------|
| 市場規模（SOM） | 8点 | SOM ≥ ¥10億: 8点, ≥ ¥1億: 5点, < ¥1億: 2点 |
| 市場成長率 | 7点 | CAGR ≥ 15%: 7点, ≥ 5%: 4点, < 5%: 1点 |
| 競争環境 | 5点 | 差別化余地大: 5点, 中: 3点, 小: 1点 |
| タイミング | 5点 | 最適: 5点, 可: 3点, 不適: 1点 |

#### 顧客課題の深刻度（25点満点）
| 項目 | 配点 | 評価基準 |
|------|------|---------|
| Pain Level | 10点 | Critical: 10点, Important: 6点, Nice-to-have: 2点 |
| WTP（支払意思） | 8点 | 高い: 8点, 中: 5点, 低い: 2点 |
| ペルソナの明確さ | 4点 | 明確: 4点, やや曖昧: 2点, 不明確: 1点 |
| 代替手段の不足度 | 3点 | 不足: 3点, 一部充足: 2点, 十分: 0点 |

#### 収益性（20点満点）
| 項目 | 配点 | 評価基準 |
|------|------|---------|
| LTV/CAC比率 | 8点 | ≥ 3x: 8点, ≥ 2x: 5点, < 2x: 1点 |
| 損益分岐点の現実性 | 7点 | ≤ 6ヶ月: 7点, ≤ 12ヶ月: 4点, > 12ヶ月: 1点 |
| 必要初期資金 | 5点 | ≤ ¥50万: 5点, ≤ ¥200万: 3点, > ¥200万: 1点 |

#### 実現可能性（15点満点）
| 項目 | 配点 | 評価基準 |
|------|------|---------|
| 1人+AIで開始可能か | 8点 | 可能: 8点, 一部外注必要: 4点, 困難: 1点 |
| 技術的ハードル | 7点 | 低い: 7点, 中: 4点, 高い: 1点 |

#### リスク許容度（15点満点）
| 項目 | 配点 | 評価基準 |
|------|------|---------|
| キルファクター | 8点 | なし: 8点, 条件付き: 4点, あり: 0点 |
| 総合リスクレベル | 7点 | 低: 7点, 中: 4点, 高: 1点 |

### Step 3: Go/No-Go判定

| 判定 | 条件 |
|------|------|
| **🟢 Go** | 総合スコア ≥ 70点 AND キルファクターなし |
| **🟡 Conditional Go** | 総合スコア 50-69点 OR 条件付きキルファクター |
| **🔴 No-Go** | 総合スコア < 50点 OR キルファクターあり（緩和不能） |

### Step 4: 判定理由と提言

- Go: 特に強いポイントと、注意すべきリスクを明記
- Conditional Go: Goに転換するための具体的条件を明記
- No-Go: 撤退すべき理由と、ピボットの可能性があれば提案

### Step 5: Phase 2移行の準備（Go/Conditional Goの場合）

Phase 2エージェントに引き継ぐべき情報を整理する：
- 最優先で取り組むべきこと
- 避けるべきリスク
- 活用すべき強み

## 注意事項

- **判定は厳格に行う**。甘い判定はユーザーの損失につながる
- 全レポートの数値を横断的にチェックし、矛盾を見逃さない
- スコアリングは機械的に行い、主観を排除する
- Conditional Goの場合、条件が具体的かつ検証可能であること
