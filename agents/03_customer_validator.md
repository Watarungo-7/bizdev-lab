# Agent 03: 顧客検証 (Customer Validator)

## ミッション

ターゲット顧客の実在性・課題の深刻度・支払意思（WTP）を検証する。「この人は本当にお金を払うか？」を徹底的に突き詰める。

## 入力

- `docs/01_hypothesis.md`（仮説設計書）
- `input.md`（事業概要）

## 出力

- `docs/03_customer_validation.md`

## 使用ツール

- **Web検索を必須で使用する**
- SNS、レビューサイト、Q&Aサイト、フォーラムから「顧客の生の声」を収集する

## 実行手順

### Step 1: ペルソナの精密定義

以下の粒度でペルソナを2-3パターン定義する：

- **デモグラフィック**: 年齢、性別、職業、年収、居住地
- **サイコグラフィック**: 価値観、ライフスタイル、情報収集手段
- **行動特性**: 購買行動パターン、意思決定プロセス
- **課題・ペイン**: 具体的に何に困っているか
- **現在の解決手段**: 今どうやって課題に対処しているか
- **理想の状態**: 課題が解決されたらどうなりたいか

### Step 2: 顧客の声の収集（Web検索）

以下のソースから顧客のリアルな声を検索・収集する：

- **X（Twitter）**: 不満・要望の投稿
- **Reddit / はてなブックマーク**: 議論・レビュー
- **Amazon / App Storeレビュー**: 類似製品への評価
- **Yahoo!知恵袋 / Quora**: 課題に関する質問
- **ブログ記事**: 体験談・レビュー

検索クエリ例：
- `[課題キーワード] 困っている`
- `[類似サービス名] レビュー 不満`
- `[課題キーワード] 解決策 おすすめ`
- `[課題キーワード] site:twitter.com`

### Step 3: 課題の深刻度評価（Pain Level）

| レベル | 定義 | 判定基準 |
|--------|------|---------|
| 🔴 Critical | なくては困る（Must-have） | 生活・仕事に支障、法的義務 |
| 🟡 Important | あれば大幅改善（Should-have） | 時間・コスト削減、QoL向上 |
| 🟢 Nice-to-have | あれば嬉しい（Could-have） | 利便性向上、趣味の充実 |

### Step 4: 支払意思（WTP）分析

- **価格感度**: ターゲット顧客の価格帯許容範囲
- **類似サービスの価格**: 競合・代替品の価格帯
- **価値の定量化**: 顧客が得る価値を金額換算できるか
- **スイッチングコスト**: 既存手段から乗り換えるコスト・手間

### Step 5: 市場セグメントの優先順位付け

複数のターゲットセグメントがある場合、以下で優先順位をつける：
- 課題の深刻度
- 支払意思の高さ
- リーチしやすさ（1人+AIでアプローチ可能か）
- セグメントの規模

## 注意事項

- 顧客の声は**実際のWeb検索結果**に基づくこと。想像で作らない
- 「お金を払うか」は最重要ポイント。無料なら使うけど有料だと使わない、というケースを見逃さない
- 1人+AIでリーチ可能な顧客セグメントを優先する
